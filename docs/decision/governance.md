# Content Governance & Freshness

Content that is accurate today silently goes stale as the product evolves. Governance is the
system that catches drift before a user (or an AI agent) gets a wrong answer.

## Framework

- **Ownership.** Every content area has a named owner accountable for accuracy.
- **Freshness policy.** Each asset has a freshness threshold; assets past it are flagged for
  review, refresh, consolidation, or retirement.
- **Quality standards.** A shared style guide and structured templates keep a federated set of
  contributors consistent.
- **Signals.** Retrieval tags and usage data drive a dashboard that makes staleness visible
  proactively, not reactively.

## Applied result

I built a BI-driven freshness system using embedded retrieval tags to flag stale assets past a
defined threshold, and a single-source content architecture with editorial standards that lifted
**response accuracy 24%** across multi-regional deployments.
