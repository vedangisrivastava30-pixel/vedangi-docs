# Content Layers Framework

A framework for structuring product content into three layers so different users can find,
understand, and act without escalating to a human.

## The three layers

| Layer | Purpose | Example |
| --- | --- | --- |
| **Orientation** | Anchor the user in a new surface or concept | "What is a unified campaign?" |
| **Decision** | Build trust for a choice: impact, risk, reversibility, source | "Should I use automated bidding here?" |
| **Implementation** | Get the task done, fast and self-serve | "Create your first campaign" |

## Why it works

Most documentation collapses these layers into one dense page. Splitting them lets each user
type enter at the right level, and it makes each block **modular and reusable** — the same
decision block can be surfaced in-product, in a guide, or retrieved by an AI assistant.

## Applied result

At Amazon Advertising, I designed this three-layer model for a platform relaunch across
Sponsored Ads and DSP. It aligned seven product managers, sales, and marketing on one content
system, shipped on time with **zero new support escalations** on covered features, and supported
a **40% awareness lift, 27% adoption increase, and $200M+ in projected annual revenue**.
