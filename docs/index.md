# Vedangi Srivastava

Senior content designer specializing in structured, AI-retrievable content architecture for
complex technical products. This site is built docs-as-code (Markdown, Git, MkDocs, GitHub Pages)
and organized the way strong developer documentation should be: by **content layer** —
orientation, decision, and implementation.

## What this demonstrates

- **Content architecture** across orientation, decision, and implementation layers
- **Structured authoring**: single-sourcing, conditional content, content modeling for reuse
- **Writing for AI retrieval**: clear hierarchy, consistent terminology, complete context per entry point
- **Content governance**: ownership, freshness policies, and quality standards at scale
- **Docs-as-code**: authored in Markdown, versioned in Git, published via GitHub Pages
